Bestia API

Shape / Position / Movement API

Erlaubt das Abfragen und setzen einer bestimmten kollidierenden Form und das Auslesen einer globalen Position auf der Welt. Erlaubt die kontrollierte, kontinuierliche Bewegung auf eine bestimmte Position. Dabei muss der Server verwendet werden, da nur ihr den Überblick über andere Entities besitzt und diese über Positionsveränderungen unterrichten muss (z.B. KI Updates).

Visual API

Ermöglicht das Abfragen einer visuellen Repräsentation der Entität. Diese Repräsentation kann geändert werden, zum Beispiel durch angelegte Items oder Statuseffekte. Diese Änderungen müssen an Clients über die Server Schnittstelle kommuniziert werden, da es sich um sichtbare und somit auch im Client vorhandene Entities handelt.

Interaction API (DRAFT)

Entities mit dieser API sind in der Lage mit dem Spieler zu interagieren. Sie zeigen auf Wunsch des Spielers ein Menü an bei dem er aus verschiedenen Optionen auswählen kann welche Interaktion er sich wünscht.

Draft: Es bleibt zu klären wie Entitäten miteinander die konkrete Interaktion aushandeln. Das heißt Menüs anzeigen etc. und dies auch autonom ermöglicht werden kann wenn z.B. nur KI Entitäten miteinander interagieren.

Mögliche Arten der Interaktion

  • Sprechen/Kontakt aufnehmen

  • Handeln

  • Angreifen

  • Untersuchen/Betrachten

Methoden

  • getPossibleInteractions(Interactable inter)

  • getInteractions()

  • triggerInteraction(Interaction inter, Interactable obj)

  • ...

Attackable

Entities die man angreifen kann müssen in der Lage sein zwecks Schadensberechnung Statuswerte liefern zu können. Auch Equipments benötigen zwangsweise Statuswerte. Weiterhin können diese Objekte Schaden nehmen, sind zerstörbar und können potentiell Statusveränderungen erleiden.

Methoden

  • getStatusvalues()

  • getOriginalStatusvalues()

  • getStatusEffects()

  • addStatusEffect()

  • removeStatusEffect()

  • takeDamage(Damage dmg) - Der Schaden wird angewendet, aber vorher per checkDamage überprüft und ggf. abgemildert. Liefert den angewendeten Schaden oder keinen Schaden (null) zurück.

  • checkDamage(Damage dmg) - Überprüfe ob der Schaden negiert oder abgemindert wird. Liefert das negierte (null) Objekt oder abgeschwächte Objekt zurück.

  • takeTrueDamage(Damage dmg) - Schaden wird nicht mehr durch Effekte reduziert.

  • kill()

Equipable API

Die Entität kann mit mit einem Ausrüstungsgegenstand ausgestattet werden. Dieser ist dann ggf. in der Lage Statuseffekte und Werte zu verändern. Die entsprechenden APIs sind also Voraussetzung hierfür. Equipments können unter bestimmten Umständen abgelegt werden und müssen daher in ein Inventar abgelegt werden können. Es muss sichergestellt werden, dass z.B. Schäden die sich auf die Entität auswirken über die Equipments abgewickelt werden können.

Methoden

  • canEquip

  • getAvailableEquipmentSlots()

  • equip(Item item)

  • takeOff(Item item)

Inventory API

Benötigt weiterhin Zugriff zur Server API um Gegenstände auf den Boden zu werfen und entsprechende Entitäten in der Umgebung zu benachrichtigen.

Methoden

  • getMaxWeight()

  • getWeight()

  • getOverweight() : Overweight

    Liefert den Grad des Overweight, dies führt dazu, dass die Bestia zunächst keine Mana regeneriert, kann bei massiven Overweight aber auch dazu führen dass die Bestia Mana/HP verliert.

  • getMaxItemCount()

  • getItemCount()

  • addItem(Item i)

  • removeItem(Item i)

  • dropItem(Item i)

  • getItems() : List
