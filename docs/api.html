Bestia API

The Bestia Browsergame will feature certain public and private APIs in order to interact with the game and its world users. There is the so called Script API which can be used inside the Bestia JavaScript files to describe item, status, attack effects etc. and the REST API which can be used via external services in order to query data from the Bestia game or to interact with it.

Script API

Movement / Position API

This API allows the reading and writing of position data or starting a movement to a certain point. It also allows to alter the collision shape of the entity which is used to determine possible collisions with other entities. The server will keep track of the entity movement and will propagate this changes in position towards the clients or other entities which need this information for example to update their AI.

Equip API

The entity can be equipped with certain items. These items add or remove status effects and change the internal states of the entity.

Equip API Methods

  • canEquip
  • getAvailableEquipmentSlots()
  • equip(Item item)
  • takeOff(Item item)

Visual API

Allows the change of the visual representation of an entity. The visual appearance can (and should be changed) in case certain effects are applied to the entity. All the changes are tracked and communicated to the connected clients.

Interaction API (DRAFT)

Entities can interact with each other or the player. The player will be shown a menu list from which he can pick the desired interaction with the entity.

Possible interactions include:

  • Speak/Contact each other
  • Trade
  • Attack
  • Search

Inventory API

The inventory API allows access to the inventory of an entity. It can add or remove items from an entity.

Inventory Methods

  • getMaxWeight()
  • getWeight()
  • getOverweight() : Overweight
  • getMaxItemCount()
  • getItemCount()
  • addItem(Item i)
  • removeItem(Item i)
  • dropItem(Item i)
  • getItems() : List
