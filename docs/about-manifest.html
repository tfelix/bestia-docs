Bestia - Manifest

A recent build of the demo version of Bestia is available under http://demo.bestia-game.net.

Dieses Dokument beschreibt die zentralen Konzepte und Designentscheidungen von Bestia. Sie enthält Übersichten über die grundlegenden Mechaniken des Spiels sowie eine Beschreibung der technischen Umsetzung. Hier finden sich auch technische Beschreibungen wie neue Inhalte zu erstellen sind. Den Richtlinien in diesem Dokument sind so präzise wie möglich Folge zu leisten. Neben diesem zentralen Dokument existieren für verschiedenste Nebenthemen sogenannte Drafts (Entwurf) Dokumente. Sobald diese ausgereift sind, werden sie in dieses Basisdokument aufgenommen. Bei allen hier aufgenommenen Informationen kann man also von einem gewissen Reifegrad ausgehen. Errate und Anpassungen sind natürlich auch in diesem, lebendigem. Dokument, nicht ausgeschlossen.

This document describes the technical details of the bestia server implementation. It should be used together with the API JavaDoc documentation in order to provide a complete description of the server system powering the Bestia game.

Das Manifest

Das Bestia Manifest beschreibt die zentralen Grundgedanken an denen sich Gameplay und Spielmechaniken orientieren sollten. Nur in absoluten Ausnahmefällen und mit gutem Grund sollte davon abgewichen werden.

Fantasy-Simulator

Wir wollen eine Geschichte erzählen. Letztendlich passiert das indem auch die Welt in einem dazu passenden Kontext gesetzt wird. Das Spiel soll die Kreativität der Spieler fördern. Nichts ist frustrierender als Rätsel durch reines Ausprobieren lösen zu können. Alle Dinge in der Welt sollen durch In-Universe Theorien sich logisch erschließen lassen. Es soll kein "magisches" User Interface geben das den Spieler wie von Geisterhand zum nächsten Dungeon teleportiert. Für so etwas soll es zum Beispiel Priester als Spielercharaktere geben die sich diesem Problem annehmen. Interaktion zwischen den Spielern soll gefördert werden.

Eine Welt, ein Server

Wir alle leben auf dem gleichen Planeten. Brauchen wir wirklich multiple Server? Mehrere Server reißen die Community auseinander. Natürlich ist es technisch herausfordernder alle Spieler durch eine gemeinsame Spielwelt zu verbinden. Dieser Herausforderung muss sich aber gestellt werden. Dadurch wird eine Interaktion der Spieler untereinander weiter gefördert.

Kein Patch-Bullshit

In modernen Spielen werden oft dauerhaft Grundlegende Änderungen der Spielmechanik vorgenommen. Dies soll vor allem die Spielerschaft durch neue Taktiken die entwickelt werden müssen bei Laune halten. Leider hängt dies Spieler ab die sich nicht tiefgreifend mit der Theorie beschäftigen wollen früher oder später ab. Lieber sollte die Art des Gameplays von Anfang an fesselnd ausgelegt werden um Langzeitmotivation sicherzustellen. Durch das konsistente Gameplay wird eine immersive Welt sichergestellt. Probleme in der Spielbalance werden natürlich nach wie vor gepatched.

Rede mit deinem Nachbarn wie mit dir selbst!

Viele Spielmechaniken sollen nur durch die Interaktion mit anderen Spielern möglich werden. Rituale die mehrere Spieler benötigen, durchaus auch die Hilfe mehrere Charakterklassen sollen eher die Norm statt die Ausnahme sein. Einzelspieler sollen natürlich auch in der Lage sein das Spiel zu genießen. Dennoch ist der Name MMORPG Programm. Das volle Potential des Spiels kann und soll man erst durch Interaktion mit weiteren Menschen erleben. Dies fördert einen Zusammenhalt in der Community der Spieler. Durch diese geförderten sozialen Kontakte werden die Spieler stärker an das Spiel selbst gebunden.

Fortgeschrittenes Item-Crafting

Items werden in der Regel nicht durch das Töten von Monstern erhalten (wir erinnern uns: Fantasy-Realismus. Tragen Wölfe Schwerter bei sich? Eher nicht…). Sie müssen hergestellt werden. Auf Maps werden in der Regel nur Rohstoffe gefunden. Fast alle Ausrüstungen werden hergestellt oder von NPCs erworben. NPC Händler haben nur einen begrenzten Vorrat an Gütern. Diese müssen erst wieder hergestellt und angeliefert werden. Dies führt zu einem Etablieren von Handelsrouten. Der Preis bestimmt sich über das Angebot und die Nachfrage. NPC sollen hier selbst versuchen eine Gewinnmaximierung zu erhalten. Spieler müssen in die Lage versetzt werden selbst möglichst leicht an diesem Warenkreislauf sowohl als Produzent als auch als Konsument teilnehmen zu können.

Die Berufe werden durch das Fähigkeiten bzw. Skillsystem des Spiels abgedeckt. Mehr dazu im Abschnitt Fähigkeiten System.

Moderne Künstliche Intelligenz

Das Spiel enthält eine moderne unglaublich leistungsfähige KI. Die Welt und deren Bewohner soll damit unglaublich lebendig wirken. Bestias und NPC gehen einem Tagesablauf nach und verfolgen eigene Ziele. Der Spieler soll durch beobachten durchaus Muster innerhalb des Ablaufes erkennen und dait interagieren können. Die NPC reagieren genauso auf Einflüsse durch den Spieler. NPCs sollen in der Lage sein auch lange zu Reisen und so ggf. auch für einige Zeit nicht für den Spieler verfügbar zu sein.

Story inside!

Die Geschichte sollte einen eigenen Humor besitzen. Durchaus übertrieben um zu zeigen dass mitnichten alles todernst genommen wird. Die Ausbildung einer Charakterklasse sollte mit einer Geschichte (Questreihe?) verbunden sein. Jede Charakterklasse kann ihre eigene Geschichte erzählen, welche der Spieler beim wählen der Klasse durchlebt (eine Ausbildung zum Krieger etc.). Es soll nicht ein Klick im Menü zum fertigen Charakter führen. Anspielungen an die moderne Popkultur sind erwünscht.

Richtlinien und Richtwerte

Gewisse Werte müssen zur Wahrung der Balance einfach innerhalb eines gewissen Rahmens bleiben. Richtlinien sollten nur in den seltensten Fällen verletzt werden um eine gute Balance zu gewährleisten. In den jeweiligen Abschnitten sind die Richtlinien beschrieben.
