Game Mechanics

Kultboni

Denkbar wäre es das besondere Landflächen unter den Einfluss eines bestimmten Kultes fallen. Dadurch werden dann bestimmte Bestias angezogen die den Kultmitgliedern freundlich gesinnt sind. Besondere Ressourcen werden generiert oder es werden verstärkt Mana-Rifts geöffnet und die Mana innerhalb des Landes steigt dort an. Weiterhin kann mit einem globalen Bonus die vorherrschende Fraktion belohnt werden. Denkbar wären zusätzliche EXP Boni.

Die dunkle Bedrohung

In jeder Welt tritt nach eine mindest Zeitspanne (mindestens 9 Monate Echtzeit) und danach mit einer gewissen Wahrscheinlichkeit eine dunkle Bedrohung auf. Dies können bedrohungen durch besonders starke Bestia sein, aber auch magische Elemente oder Naturkatastrophen wie Meteoriten. Unter bestimmten Bedingungen kann dieses Ereignis ebenfalls zur Zerstörung der Welt beitragen. Sollte die Zerstörung durch das Event herbeigerufen werden dann profitieren keine der Kulte davon. Man muss sich also durch dieses Event in einer gewissen Form zusammenschließen.

Charakterklassen

Alle Charaktere starten als Novizen mit einem gewissen Grundset an Fähigkeiten die sie nutzen können um in der Welt zu überleben. Zum Beispiel können alle Charaktere Fähigkeiten verwenden um Gegenstände herzustellen. Diese erlernten Fähigkeiten gehen auch nicht beim Wechsel in eine andere Charakterklasse verloren.

Im Verlauf des Spiels kann über eine Questreihe in eine andere Charakterklasse gewechselt werden. Bei der Erstellung der Welt muss darauf geachtet werden, dass diese Quests immer durchführbar bleiben.

Fähigkeiten

Die Fähigkeiten (Skills) eines Charakters ermöglichen ihm zielgerichtet seine Fertigkeiten auszubauen. Das Skillsystem ist ein hybrides System welches sich aus klassischem verteilen von Punkten ergibt, aber auch durch Training und Ausführen von Fertigkeiten erlernt werden kann.

Skills können abhängig voneinander sein, sodass man zunächste diverse Voraussetzungen erfüllen muss um Skillpunkte in bestimmte Skills investieren zu können.

Die Skills sind rein passiver Natur und ermöglichen es lediglich den Hauptcharakter bzw. die Bestias feiner zu differenzieren. Die Attacken/Skills die ein Spieler einsetzen kann erlernt die Bestia separat entweder über Levelaufstiege oder spezielle Schriftrollen oder Training.

Allgemeine Richtlinien

Fähigkeiten sollten wenn mögliche eine gewisse Interaktion mit dem Spieler benötigen von deren Erfolg die Wirkung der Fähigkeit abhängt. Ein Beispiel wäre die Teleport Fähigkeit: nach Auswahl der Spieler (welche flexibel von der Zahl sein kann) und der magischen Interferenz am Zielort muss der Spieler ein kleines Minispiel im Client gewinnen wie ein Balken in einem bestimmten Bereich zum stehen bringen. Abhängig davon wie erfolgreich dies war wird der Spieler dann präziser zum Zielort transportiert.

Letztlich bleibt es wichtig den Ausgang von Fähigkeiten in gewisser Weise offen zu lassen. Dabei sollte das Ergebnis aber durch den Spieler beeinflussbar bleiben.

Erfahrung

Der primäre Weg Erfahrung zu erlangen ist durch den Kampf mit gegnerischen Bestias. Es gibt aber gerade für pazifistishcere Spieler ebenfalls noch Erfahrung zu sammeln durch den Einsatz von Skills. Um aber Spieler keinen Vorteil gegenüber kämpfenden Spielern zu geben, wird die Erfahrung durch Skills in einem anderen Pool verrechnet. Überschreitet der Anteil dieser EXP am Levelup einen Anteil von 65% gilt das Levelup als "Job" Level Up. Dann können die erworbenen Stat Up Punkte nicht in offensive Fähigkeiten wie INT oder STR investiert werden. Wird ein normales Level Up vorgenommen können die Punkte vollkommen normal verteilt werden.

Jedes Level Up ergibt eine gewisse Menge an Effort Points die über die Gesamtheit der Statuspunkte vom Spieler nach Belieben verteilt werden können. Weiterhin ergibt jedes Level Up unterhalb von Level 50 einen Punkt für ein Skill Up. Ab Level 50 nimmt die Anzahl der vergebenen Skill Up Punkte langsam ab. Sodass man unter umständen mehrere Level Ups benötigt um einen weiteren Skill Up zu erhalten.

NPCs

Wie wird ein achtloser Umgang mit NPCs verhindert?

Da es möglich ist NPCs zu töten und dies eine nachhaltige Beeinflussung des Spielewelt zur Folge hat muss der "mordende" Spieler mit Konsequenzen belegt werden um solches Verhalten zu bestrafen. Freundschaftlicher Umgang mit den NPCs hingegen muss belohnt werden.

Ehre-System

Jeder NPC führt eine interne Liste mit Reputationspunkten und Spielern. Sollte sich ein Spieler daneben oder freundlich benehmen wird diese Liste angepasst. Der NPC kommuniziert diese Liste auch mit anderen NPCs sollten sie sich treffen. Sollten die Reputationen bestimmte Ober- und Untergrenzen überschreiten und kommt der NPC in Gebiete die ein weitreichendes Kommunikationssystem nutzen so kann es sein, dass der NPC dies auch in entferntere Gebiete kommuniziert. Nur wenn die Reputation einen bestimmten Wert unter oder überschreitet merkt sich der NPC dies auf Dauer. Andere Werte gleichen sich langsam wieder dem Standardwert an und werden dann aus der Liste entfernt (NPCs vergessen also nicht wenn ein Spieler Familienmitglieder getötet haben oder das NPC Leben gerettet wurde). Erreicht die Reputationspunkte nach einer gewissen Zeit die nächstgelegene Siedlung wird dort der Wert auf Dauer gespeichert.

Sinkt die Reputation unter einen gewissen Wert können Kopfgelder auf den Spieler ausgesetzt werden und Städte werden sich aktiv gegen den Spieler zur Wehr setzen. NPCs auf dem Land werden ihn angreifen oder die Flucht antreten.

Steigt die Reputation werden sich die NPC generöser beim Erfüllen von Aufgaben zeigen oder dem Spieler sogar manchmal einfach so kleine Geschenke machen.

Reputation sinkt durch böse Taten schneller als durch gute erarbeitet. Es gibt einige wenige Punkte im Spiel um an Gottesschreinen ein Opfer darzubringen eine Reputation die unter die Schwelle zur globalen Verfolgung eines Spielers geraten ist anzuheben. Das Opfer ist in aller Regel (je nach Reputationsschwelle) sehr schmerzhaft und selbst wenn es erbracht wurde kann es mehrere Tage dauern, bis die Kunde darüber verbreitet wurde.

Erfolge

Es sollte eine Reihe von Erfolgen geben die dem Spieler entsprechend durch ein erscheinendes Fenster angezeigt werden. Erfolge können auf der Master-Profilseite angezeigt werden.

Bestimmte einzigartige Erfolge beziehen sich auf eine Welt. Diese sollten in jeder neuen Welt resettet und von anderen Spielern ebenfalls erreicht werden können. Das Datum + den Namen der Welt in der der Erfolg erzielt wurde bleibt für einen Spieler jedoch immer in der Historie.

Spielmechaniken

Das Spiel bietet unter berücksichtigung der immer geltenden Manifest aussagen mehrere Hauptaspekte des Gameplays, die der Spieler wahrnehmen kann. Darüber hinaus soll sich durch das Konzept der offenen welt ein emergente Spielprinzipien auftun.

Hier werden konkrete Realisierungen von zentralen Spielelementen beschrieben. Im Gegensatz zu den groben Richtlinien des Manifestes geht es hier durchaus schon um die Diskussion und Beschreibung eines konkreten Spielfeatures. Dabei sollten die Ausprägung des jeweiligen Features so genau wie möglich beschrieben werden um eine leichte Implementation zu ermöglichen. Die Liste wird stetig angepasst bzw. erweitert.

Erforschung der Welt

Ein Großteil der Spielwelt wird automatisch generiert. Es sollte eine Anzahl an Spieler geben die sich mit der Erforschung der Welt befasst. Dabei können neue Gebiete kartografiert werden und einzigartige Ressourcen entdeckt und erforscht werden. Generell sollte sich ein eigener Zweig der Spieler über die Geheimnisse der Welt auslassen, um neue Items zu entdecken bzw. wie diese geschaffen werden. Die gesamte Welt muss nach und nach von Spielern erkundet und Kartographiert werden. Dazu muss regelmäßig eine verdeckte Karte aufgedeckt werden. Die Spieler erhalten die Möglichkeit diese Karte online einzusehen und können so den Gesamtfortschritt der Erkundung wahrnehmen.

Mögliche Events können dann auch auf dieser öffentlichen Karte angekündigt werden. Jeder Spieler sollte noch im Spiel eine Weltkarte erstellen können. Spezielle Kartographierer sollten in der Lage sein diese Karten zu kombinieren oder neue, magische Karten zu generieren die dann über mögliche Ereignisse informieren (Informationsaustausch mit Freunden über Marker, oder Anzeige bestimmter Rohstoffquellen).

Items und Entities Interaktion

Es sollte möglich sein Items mit Entities zu verknüpfen. Eine Verschachtelung sollte also möglich sein. Mit dieser Technik wird es erlaubt aktive Items zu kombinieren. Zum Beispiel:

  • Energiekristalle an ein magisches Artefakt anbringen bzw. entnehmen.

  • Kisten die mit Inhalt ausgestattet werden können.

Zu diesem Zwecke müssen die Items limitiert werden können die in eine Entity gelagert werden können. Das Filtern muss sehr flexibel sein, bestimmte Items ausschließen können, das Gewicht limitieren können etc. Jeder Gegenstand besitzt auch eigene Widerstandswerte. Sobald eine Bestia getötet wird wird überprüft ob und wenn ja wieviel Schaden einzelne, getragene Gegenstände nehmen. Auch Gegenstände die auf dem Boden liegen können durch Attacken angegriffen und ggf. zerstört werden. Items aus Papier sind zum Beispiel anfälliger als Rüstungsteile aus Metall.

Zerstörung von Items

Sogut wie jedes Item in Bestia kann beschädigt werden. Es wird entsprechende Berufe geben die in der Lage sind Gegenstände wieder zu reparieren. Gegenstände funktionieren in einem Weiten Rahmen ihres Zustandes normal. Ab einem gewissen Grad verlieren sie langsam ihre Eigenschaften (Rüstungen ihren Rüstwert, Waffen treffen seltener, Magische Stäbe erleiden einen Schadensmalus usw.). Bis die Gegenstände unbenutzbar werden. Wenn man ihnen dann weiter Schaden zufügt können sie komplett und unwiederbringlich zerstört werden.

Spielertod

Spieler selbst können nicht sterben, sie werden mit einem temporären Malus wiedergeboren. Bestias hingegen können durch besonders schwere Treffer komplett getötet werden. Aktiviert der Spieler den sogenannten "Hard Mode" bekommt er einen Erfahrungs- sowie Lootbonus. Er verliert jedoch bei einem Tod alle seine Gegenstände und einen gewissen Teil seiner Erfahrung. Im “Soft Mode” verliert der Spieler keine Gegenstände. Seine Gegenstände erleiden jedoch innerhalb seines Inventars eine Beschädigung und er verliert weniger Erfahrung. Dieser “Soft Mode” wird mittels eines besonders, für Bestia Master vorbehaltenen Zauber erklärt. Es handelt sich um einen Zauber, das heißt er kann theoretisch durch besondere Anti-Magie umgangen werden (!).

Der Spieler kann nicht beliebig wechseln. Sollte er einmal den "Hard Mode" aktiviert haben und sich entscheiden in zu deaktivieren, so wird dieser Prozess eine Stunde Echtzeit in Anspruch nehmen. Dies soll verhindern dass Spieler den Modus nach Erkennen einer brenzligen Situation schnell ein- und ausschalten.

Postsystem

Angelehnt an World of Warcraft soll ein Postsystem etabliert werden. Das System soll das Versenden von Briefen, vor allem aber Gegenständen und Geld an andere Spieler und Orte ermöglichen. Es soll aber anders als in den meisten Spielen kein "out-of-world" System sein. Denkbar wäre dass Spieler das System selbst organisieren müssen und so Bestia als Postboten in entlegene Orte Aventerras schicken. Dies soll die Geschwindigkeit aus dem Spiel nehmen. Auf Eintreffen muss länger gewartet werden und der Aufwand für die Spieler steigt.

Denkbar wäre auch ein kostenpflichtiges Telegramm System für Spieler um Nachrichten mit wertvollen Gütern schneller und sicherer zu transportieren (kostet dann aber wesentlich mehr Gold). Sehr schön wäre ein reller Transport der Post im Spiel mit Hilfe von einem oder mehreren NPC. Dieser könnte dann wie Spieler selbst Opfer von Überfällen und Wegelagerern werden. Es ist nur nicht sicher ob die KI solche Aufgaben zuverlässig zu leisten vermag.

Brieftauben, Magisches Postsystem (ressourcen Intensiv), Zustellzeit abhängig von der Distanz zum Ziel

Manasystem

Mana durchdringt die Welt. Die Konzentration der Manaströme wird in jeder Inkarnation des Bestias Spieles erneut bestimmt. Die Manakonzentration dient unter anderem dazu, wie schnell die Bestia in einem bestimmten Gebiet Mana regenerieren. Mana ist ein sehr zwiegespaltener Aspekt. In Arealen mit hoher Konzentration lassen sich magische Artefakte erbauen, mächtige Rituale abhalten und Mana "ernten" um es zu Forschungszwecken zu transportieren. Es lockt aber auch mächtige Bestias an und ab einer hohen Konzentration gibt es magische Stürme, Entladungen und ggf. sogar Rift-Events die Weltbosse hervorbringen.

Unvorhergesehene Ereignisse

Zufallsgenerator löst zufällige, spannende Ereignisse innerhalb der Welt aus. Dabei können die Ereignisse durchaus Teile der Weltkarte nachhaltig beeinflussen oder verändern. Zum Beispiel können Pfade oder ganze Landschaften unbenutzbar werden oder neue Wege erschlossen werden. Mögliche Ereignisse sind:

  • Meteoriten Einschlag

  • Extreme Wetterereignisse (Stürme über mehrere Tage)

  • Vulkanausbrüche

  • Tsunami / Überflutungen

  • Waldbrände

  • Öffnen eines Weltrifts und massiver Mana Anstieg mit allen damit verbundenen Konsequenzen

  • Auftauchen von sehr starken einzelnen oder mehreren Bestias, sogenannte Weltbosse

Natürlich können nicht alle Ereignisse an allen Orten auftauchen. Manche Ereignisse sind abhängig vom Biom ob sie dort überhaupt auftreten können.

Jahreszeiten

Wechselnde Jahreszeiten sollen dazu führen einen Eindruck der Veränderlichkeit der Welt zu gewinnen. Man muss sich auf periodisch ändernde Umwelt einstellen. Dinge die im Sommer funktionieren sind vielleicht im Winter nicht mehr ohne weiteres oder gar nicht mehr möglich. Damit die Spieler aber kein reales Jahr warten müssen vergeht die Zeit in Bestia schneller.

Ein Jahr in Bestia entspricht 8 Monate in Echtzeit. Zwei Monate in Echtzeit sind also eine komplette Jahreszeit im Spiel (3 Monate im Spiel). Da es in Bestia nur einen Server Weltweit gibt sollten alle Spieler einen Tag und Nachtwechsel erleben können. Ein Bestia Tag dauert daher auch nur 16h.

  • Bestia Jahr: 8 Monate

  • Sommer, Winter, Herbst, Frühling: je 2 Monate

  • Bestia Tag in Echtzeit: 16h

Erweiterte Künstliche Intelligenz

Dieses Dokument beschreibt die KI welche bei Bestia Behemoth zum Einsatz kommt. Es erläutert die Grundlegenden Verhaltensweisen der Bestias und NPCs welche durch Algorithmen abgebildet werden sollen.

Das KI System in Bestia soll neue Maßstäbe setzen. Bestias und NPCs gleichermaßen sollen in der Lage sein langfristige Pläne, so genannte Game-Plans oder kurz GPs, zu erstellen. Sie folgen dann dieser Strategie bis Beobachtungen oder Einflüsse aus der Spielwelt sie zur Aufgabe dieses Plans zwingen oder der Plan erfüllt wurde.

Bestias suchen nach Wegen ein möglichst hohes inneres Glück zu erhalten. Dazu müssen sie schlafen, essen und fühlen sich von Manaströmen angezogen. Natürlich versuchen sie Bedrohungen aus dem Weg zu gehen und ihre Gesundheit zu schützen. Hier unterscheiden sich einige Subtypen voneinander. Manche neigen eher zu aggressivem Verhalten und greifen Spieler und andere Bestias aktiv an, andere hingegen sind eher scheu und ergreifen schnell die Flucht. Es kann aber durchaus auch Vertreter geben die, wie man sagen würde, "dumm" sind und Spieler bis zum Tode angreifen werden sobald einer in Sichtweite gerät.

Die Entscheidungen sollen aufgrund einer Wissensgrundlage getroffen werden. Ein Sinnes Simulation soll genutzt werden die Welt zu verstehen und aufgrund dessen Entscheidungen zu fällen.

Die grundlegenden KI Formen sollen per "Shortcut" in der Mob Tabelle (und für NPCs im Map Editor) zuweisbar sein. Für Bosse oder spezielle Bestias sollen aber Scripts abgelegt werden können um zielgerichtetes Verhalten zu ermöglichen.

Bestia Zucht

Folgende Eigenschaften lassen sich durch Zucht oder Paarung durch den Spieler verändern. Sicher lassen sich auch Mechaniken ausdenken welche noch nachträgliche, gewisse Anpassungen erlauben. Mögliche Anpassungen wären:

  • Attacken Liste

  • IVs

Veränderliche Welt

Nach jedem Rift Event wird die Welt von Bestia automatisch neu generiert. Die Welt soll dabei größer werden. Das Rift Event soll schwerer herbei zuführen sein. Theoretisches Limit? Das Event soll sich ankündigen durch entsprechende Katastrophen. Es muss eine Kraft geben die versucht das Rift Event zu verhindern (NPCs?). Die Spieler müssen es versuchen einzuleiten (vielleicht gibt es auch Spieler die es verhindern wollen). Danach wird die Welt neu zufällig berechnet. Es sollen hier verschiedene Klimazonen erstellt werden. Die großen Städte werden Random per Template eingefügt. Große Wasserflächen. Spieler können damit noch mehr Einfluss auf die Welt nehmen (Schnee, Regen etc.) und ggf. auch die Klimazonen verändern.

Komplexe Entities

Dynamische Objekte sind im Prinzip Entities die mehr oder weniger statisch auf der Karte positioniert werden. Diese Objekte setzen sich aber dennoch oft aus einer Komposition von mehreren, komplexeren Datenquellen zusammen. Es können so zum Beispiel mehrere Sprites kombiniert werden und dazu Licht bzw. Partikeleffekte hinzugefügt werden.

Die Beschreibungen müssen vom Server herunter geladen werden können und das Objekt danach erstellt werden.

Die Datensätze befinden sich im Assetordner: /objects

Ein Beispieldatensatz sieht so aus:

data: [

{type: 'particle', name: 'sparks', data: {count: 10, minScale: 0.1, maxScale: 0.3}},

{type: 'sprite', name: 'bonfire_small'},

{type: 'filter', name: 'heat'}

],

template: [

{ref: 'fire', name: 'bonfire_small', animation: 'burning'},

{name: 'filter', target: 'fire', params: {size: 10}},

{name: 'particle', target: 'fire', x: 10, y: 15}

]

}

Der Data Abschnitt definiert die genutzten Daten. Es kann sich dabei einerseits um Ressourcen handeln die extern nachgeladen werden müssen, wie zum Beispiel Sprites oder Sounds. Es die Daten können dort aber auch direkt beschrieben sein. Wie zum Beispiel eine Beschreibung für ein Partikelsystem.

Nachdem die Beschreibung geladen wurde, wird sie an eine Factory übergeben. Diese baut anhand des data Abschnittes eine Ressourcen Datenbank und ein Asset Pack auf. Das Asset Pack wird einem Loader übergeben. Sobald der Ladevorgang abgeschlossen ist, kann über die Factory ein entsprechendes Objekt erzeugt werden. Die Factory nutzt dabei die Informationen die auch in ihrer Ressourcen Datenbank abgelegt wurden. Die Angaben im Template werden nach und nach abgearbeitet um das finale Objekt dann zu erstellen. Einträge die die Factory nicht kennt können ignoriert werden.
